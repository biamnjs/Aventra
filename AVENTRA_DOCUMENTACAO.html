1. Visão Geral do Projecto

A Aventra é uma plataforma web de viagens personalizadas com Inteligência Artificial. O utilizador cria um perfil de viajante (preferências musicais, actividades, orçamento, clima) e a IA gera recomendações de destinos, roteiros dia a dia e playlists musicais adaptadas a cada viagem.

Cor Brand

Laranja #ff5a2c — usado em botões, destaques e favicon

Idioma da Interface

Português de Portugal — todo o texto ao utilizador está em PT-PT

Assistente Virtual

Ava — chat flutuante disponível em todas as páginas quando autenticado

Modelo IA

Claude Sonnet 4.6 via Anthropic API (claude-sonnet-4-6)

Funcionalidades Principais

  • Registo e autenticação com JWT (7 dias)
  • Questionário multi-step de perfil do viajante (6 passos)
  • Recomendações de destinos personalizadas por IA
  • Gestão de viagens (criar, actualizar, eliminar, mudar status)
  • Geração de roteiro dia a dia por IA para cada viagem
  • Playlists musicais geradas por IA com seleção de 16 géneros
  • Sistema de favoritos (destinos, hotéis, restaurantes, etc.)
  • Chat com assistente Ava em linguagem natural
  • Exploração de 12 destinos pré-populados com hotéis e avaliações

10. Como Iniciar o Projecto (Sessão de Desenvolvimento)

⚠️ Executar SEMPRE nesta ordem ao início de cada sessão O SSH tunnel não persiste entre reinícios do computador ou fecho de terminal.

Passo 1 — Abrir o túnel SSH para a base de dados

ssh -i C:/Users/Asus/.ssh/id_ed25519 -f -N -L 5433:localhost:5433 root@187.124.39.3

Este comando corre em background (-f -N). Só precisas de correr uma vez por sessão.

Passo 2 — Iniciar o Backend

cd C:\Users\Asus\Desktop\Aventra\backend
npm run dev

O backend inicia em http://localhost:3001. Usa nodemon — reinicia automaticamente quando alteras ficheiros .ts.

⚠️ Atenção Se alterares o .env, o nodemon não reinicia automaticamente — tens de matar o processo e correr npm run dev de novo.

Passo 3 — Iniciar o Frontend (noutro terminal)

cd C:\Users\Asus\Desktop\Aventra\frontend
npm run dev

Frontend disponível em http://localhost:5173. O Vite faz proxy das chamadas /api/* para localhost:3001.

Verificar se tudo está a funcionar

# Verificar backend
curl http://localhost:3001/health

# Verificar base de dados (deve retornar destinos)
curl http://localhost:3001/api/v1/destinations/featured

Forçar paragem do backend (se porta 3001 em uso)

Correr no PowerShell:

Get-NetTCPConnection -LocalPort 3001 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }

11. Problemas Conhecidos e Soluções

ProblemaCausaSolução
"Erro interno do servidor" em todos os endpoints SSH tunnel fechado — BD inacessível Correr o comando SSH tunnel (Passo 1)
Backend não reinicia após alterar .env Nodemon só observa ficheiros .ts Matar processo e reiniciar manualmente
Login não mostra erro, apenas recarrega Interceptor 401 fazia redirect mesmo sem token ✅ Corrigido — só redireciona se havia token activo
"Credenciais inválidas" no login Conta criada quando BD estava offline — não foi guardada Registar de novo em /registar
IA retorna 503 "Serviço temporariamente indisponível" CLAUDE_API_KEY inválida ou sem créditos Verificar chave no .env e adicionar créditos em console.anthropic.com
@types/express v5: req.params como string | string[] Breaking change no v5 ✅ Usar sempre req.params['id'] as string
Prisma JSON fields rejeitam unknown[] Tipo incompatível com InputJsonValue ✅ Cast: data as Prisma.InputJsonValue[]
pkill não mata processos Node no Windows pkill é comando Linux ✅ Usar PowerShell: Get-NetTCPConnection | Stop-Process

Decisões Técnicas Importantes

DecisãoRazão
Node.js + Express em vez de Hono.jsPedido explícito (conforme PDF de especificação)
BD no VPS em vez de localSem PostgreSQL nem Docker instalados na máquina de desenvolvimento
JWT_EXPIRES_IN hardcoded como '7d'jwt.sign() com variável de ambiente falha nos tipos do TypeScript
Tailwind v4 com @tailwindcss/viteUsa @import "tailwindcss" e @theme {} — sintaxe diferente do v3
Seed com IDs derivados do nomePermite upsert idempotente (re-executar seed não duplica dados)
Rotas /destinos públicasMelhor para SEO e partilha de links
